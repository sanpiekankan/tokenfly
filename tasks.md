# tokenfly 开发任务清单

## 1. 文档说明

本文档基于 [requirement.md](file:///Users/miaozhihuang/company_project/tokenfly/requirement.md) 和 [dev-plan.md](file:///Users/miaozhihuang/company_project/tokenfly/dev-plan.md) 拆解，目的是将开发计划进一步细化为可执行任务清单，便于排期、分工、跟踪和验收。

任务状态建议：

- `[ ]` 未开始
- `[-]` 进行中
- `[x]` 已完成

优先级说明：

- `P0` 必须优先完成，阻塞后续主链路
- `P1` 高优先级，直接影响 MVP 交付
- `P2` 中优先级，属于增强能力
- `P3` 长期规划项

## 2. 当前目标

当前建议先聚焦 MVP，因此任务优先围绕以下目标展开：

- 完成 Monorepo 基础工程
- 抽离 `@tokenfly/core`
- 保留并升级现有 CLI
- 增加模型价格配置与成本估算
- 为 `router` 和 `budget` 预留统一接口

## 2.1 通用技术实现要求

- [ ] `P0` 工程语言统一为 TypeScript，源码放在各模块的 `src/` 目录，构建产物输出到 `dist/`
- [ ] `P0` 使用 `npm workspaces` 管理 Monorepo，根 `package.json` 必须设置 `private: true`
- [ ] `P0` 构建工具统一使用 `tsup` 或等价轻量构建工具，输出 `esm + cjs + d.ts`
- [ ] `P0` 测试框架统一使用 `vitest`，命令入口在根目录统一编排
- [ ] `P0` 所有对外导出函数必须补充函数级注释，注释内容至少包括参数、返回值、异常或边界行为
- [ ] `P1` 配置文件优先采用 JSON 结构，运行时支持传入 JS 对象覆盖默认配置
- [ ] `P1` 所有模块必须遵循统一错误结构，至少包含 `code`、`message`、`details`
- [ ] `P1` 所有模块必须避免直接耦合具体供应商 SDK，应通过 adapter 或 provider 接口隔离
- [ ] `P1` 所有关键路径必须具备可观测性埋点，至少记录耗时、输入规模、结果摘要和错误信息
